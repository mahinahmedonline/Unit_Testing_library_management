# Library Management System - Unit Testing Report

## Objective
This repository contains the comprehensive **Unit Testing Report** for the Library Management System. The primary goal of this testing phase is to validate the individual units/components of the software to ensure they perform precisely as designed under various conditions.

## SQA Focus
As a Software Quality Assurance (SQA) engineer, this testing effort emphasizes:
- **Ensuring Code Quality:** Validating that every discrete function of the library management logic (e.g., adding books, checking out, calculating fines) behaves correctly.
- **Early Defect Detection:** Identifying and resolving logical and functional bugs at the component level before integration.
- **Robustness:** Testing edge cases, boundary values, and handling of invalid inputs (e.g., negative book quantities, invalid user IDs).

## Contents
- [`Unit_Testing_library_management testing report.pdf`](./Unit_Testing_library_management%20testing%20report.pdf): The detailed testing documentation containing test cases, execution logs, defect reports, and coverage metrics.

## Testing Scope
The unit testing covers the core modules of the Library Management System:
1. **User Management:** User registration, authentication, and role validation.
2. **Catalog Management:** Adding, updating, and removing books/media.
3. **Transaction Processing:** Borrowing, returning, and reserving items.
4. **Fine Calculation:** Logic for overdue penalty estimation.

## Testing Methodology & Metrics
- **Test Design Technique:** Boundary Value Analysis (BVA) and Equivalence Partitioning (EP).
- **Coverage:** Aimed for high branch and statement coverage.
- **Defect Tracking:** All found discrepancies during the unit test phase are documented with their severity, status, and steps to reproduce.
